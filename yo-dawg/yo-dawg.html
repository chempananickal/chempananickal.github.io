📖 WHAT IS A DAWG???!

A Directed Acyclic Word Graph (also called a Suffix Automaton) is a minimal finite automaton that accepts every suffix of a word — and therefore every substring. It is the smallest possible structure that can recognise all substrings of a word.

Unlike a trie, which stores each suffix separately, the DAWG merges states that accept the same set of future suffixes into a single node. The result has at most 2n − 1 states and 3n − 4 transitions for a word of length n.

What can you do with it? After building the DAWG for string S, you can find the Longest Common Substring between S and any query string T in O(|T|) time by traversing the automaton.

🧠 Key Concepts

State
A node. Each state represents an equivalence class of substrings that end at the same set of positions in the original word.
Transition
A labelled directed edge. Following edge labelled c from state u means "I just read character c".
Suffix Link
A dashed edge from state u to the state representing the longest proper suffix of u's equivalence class. Used to backtrack.
len(u)
The length of the longest string in state u's equivalence class.

sa_extend(last, c):
    cur      = new_state()
    cur.len  = last.len + 1
    p        = last
    while p != NULL and p.next[c] is undefined:
        p.next[c] = cur        # walk up, add transitions
        p = p.link
    if p == NULL:
        cur.link = root        # no existing c-path found
    else:
        q = p.next[c]
        if p.len + 1 == q.len:
            cur.link = q       # q is already minimal
        else:
            clone        = copy(q)
            clone.len    = p.len + 1
            while p != NULL and p.next[c] == q:
                p.next[c] = clone   # redirect existing paths
                p = p.link
            q.link   = clone
            cur.link = clone
    last = cur
